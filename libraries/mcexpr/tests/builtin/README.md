# 内置函数测试 (builtin)

## 概述

本目录包含表达式引擎内置函数的测试用例。内置函数是表达式引擎提供的标准函数库，包括数学函数、字符串函数和类型转换函数。

## 测试文件

- `test_math.cpp`: 数学函数测试 (6个测试用例)
- `test_string.cpp`: 字符串函数测试 (6个测试用例)
- `test_conversion.cpp`: 类型转换函数测试 (4个测试用例)

**总测试用例数**: 16 个

## 测试文件说明

### 1. test_math.cpp - 数学函数测试

测试表达式引擎提供的数学函数：

#### AbsFunction - 绝对值函数
- 测试正整数、负整数、正浮点数、负浮点数、零
- 测试参数错误（无参数、字符串参数）
- 测试多余参数（允许但不使用）

#### MinFunction - 最小值函数
- 测试整数、浮点数、混合类型
- 测试单个参数
- 测试参数错误（无参数、字符串参数）

#### MaxFunction - 最大值函数
- 测试整数、浮点数、混合类型
- 测试单个参数
- 测试参数错误（无参数、字符串参数）

#### PowFunction - 幂运算函数
- 测试整数幂、浮点数幂、负指数、零指数
- 测试参数错误（无参数、参数类型错误）

#### SqrtFunction - 平方根函数
- 测试整数、浮点数、零、负数（错误情况）
- 测试参数错误（无参数、字符串参数）

#### FloorFunction - 向下取整函数
- 测试正浮点数、负浮点数、整数、零
- 测试参数错误（无参数、字符串参数）

### 2. test_string.cpp - 字符串函数测试

测试表达式引擎提供的字符串处理函数：

#### LengthFunction - 字符串长度函数
- 测试空字符串、普通字符串、包含特殊字符的字符串（如中文）
- 测试参数错误（无参数、数值类型）
- 测试多余参数（允许但不使用）

#### ConcatFunction - 字符串连接函数
- 测试无参数（返回空字符串）、单个字符串、多个字符串
- 测试混合类型（字符串、整数、浮点数、布尔值）
- 测试多种类型混合连接

#### SubstringFunction - 子字符串函数
- 测试基本子字符串提取、从中间提取、提取到末尾
- 测试边界情况（起始位置为0、长度为0、超出范围）
- 测试参数错误（无参数、参数类型错误、参数个数错误）

#### UpperFunction - 转大写函数
- 测试小写字符串、混合大小写、全大写、空字符串
- 测试包含数字和特殊字符的字符串
- 测试参数错误（无参数、非字符串参数）

#### LowerFunction - 转小写函数
- 测试大写字符串、混合大小写、全小写、空字符串
- 测试包含数字和特殊字符的字符串
- 测试参数错误（无参数、非字符串参数）

#### TrimFunction - 去除首尾空白字符函数
- 测试前后空白、仅前空白、仅后空白、无空白
- 测试仅空白字符、空字符串
- 测试参数错误（无参数、非字符串参数）

### 3. test_conversion.cpp - 类型转换函数测试

测试表达式引擎提供的类型转换函数：

#### ToStringFunction - 转换为字符串函数
- 测试整数（正数、负数）、浮点数、布尔值（true、false）
- 测试已经是字符串的情况（保持不变）
- 测试参数错误（无参数）
- 测试多余参数（允许但不使用）

#### ToBoolFunction - 转换为布尔值函数
- 测试数值（1、42、0、3.14、0.0）
- 测试字符串（'true'、'yes'、'1'、'on'、'enabled'、'false'、'no'、'0'、'off'、'disabled'、空字符串）
- 测试已经是布尔值的情况（保持不变）
- 测试参数错误（无参数、无法转换的类型）

#### ToIntFunction - 转换为整数函数
- 测试字符串（'42'、'-42'、'3.14'、'true'、'false'）
- 测试浮点数（3.14、-3.14、0.0）
- 测试布尔值（true、false）
- 测试已经是整数的情况（保持不变）
- 测试参数错误（无参数、无法转换的字符串）

#### ToDoubleFunction - 转换为浮点数函数
- 测试字符串（'3.14'、'-3.14'、'42'、'-42'）
- 测试整数（42、-42、0）
- 测试布尔值（true、false）
- 测试已经是浮点数的情况（保持不变）
- 测试参数错误（无参数、无法转换的字符串）

## 运行测试

### 使用 Meson 运行

```bash
# 在项目根目录下
cd builddir
meson test expr/builtin
```

### 运行特定测试文件

```bash
# 运行所有内置函数测试
meson test expr/builtin

# 运行特定测试用例
meson test expr/builtin --gtest_filter="math_builtin_test.AbsFunction"
```

### 使用 GTest 直接运行

```bash
# 编译后直接运行测试程序
./builddir/tests/libmcpp_test --gtest_filter="*builtin*"
```

## 测试原则

1. **全面性**: 覆盖所有内置函数的正常使用和错误情况
2. **类型安全**: 测试各种输入类型的处理
3. **边界值**: 测试边界值和特殊值（如零、空字符串）
4. **错误处理**: 测试各种错误输入的处理

## 注意事项

1. 所有内置函数都注册在全局上下文中，测试使用 `builtin::get_instance().get_context()` 获取上下文
2. 某些函数允许多余参数（参数个数大于形参个数），但只使用前 N 个参数
3. 类型转换函数会尝试将输入转换为目标类型，如果无法转换会抛出异常
4. 字符串函数对中文等多字节字符的支持取决于底层字符串实现
