# ServiceFactory 服务工厂

## 1. 概述

服务工厂(ServiceFactory)是libmcpp框架中负责服务注册和创建的核心组件。它实现了工厂模式，允许应用程序动态注册服务类型，并在需要时创建这些服务的实例。

## 2. 主要功能

- **服务类型注册**：将服务类型与其名称关联
- **服务实例创建**：根据服务类型名称创建服务实例
- **服务选项管理**：管理服务的命令行和配置选项
- **服务类型查询**：提供服务类型的存在性检查和列表获取

## 3. 类定义

```cpp
namespace mc {
    class service_factory {
    public:
        struct service_options {
            po::options_description cli;  // 命令行选项
            po::options_description cfg;  // 配置文件选项
        };

        // 构造和析构
        service_factory();
        virtual ~service_factory() = default;

        // 注册服务类型
        template <typename ServiceType>
        void register_service(const std::string& service_name);

        // 创建服务实例
        virtual service_ptr create_service(const std::string& service_name, 
                                           std::string object_name,
                                           mc::dict args);

        // 检查服务类型是否存在
        bool has_service(const std::string& service_name) const;

        // 获取所有注册的服务类型
        std::vector<std::string> get_service_types() const;

        // 获取服务选项
        std::unique_ptr<service_options>& get_service_options();

    private:
        std::unordered_map<std::string, service_creator> m_creators;
        std::unique_ptr<service_options> m_opts;
    };
}
```

## 4. 工作流程

1. **服务注册**：模块通过`register_service<ServiceType>()`注册服务类型
2. **选项注册**：服务类型可以注册自己的命令行和配置选项
3. **服务创建**：通过`create_service()`创建服务实例
4. **初始化**：新创建的服务使用提供的参数进行初始化

## 5. 使用示例

```cpp
// 注册服务类型
service_factory factory;
factory.register_service<DatabaseService>("database");
factory.register_service<LogService>("logger");

// 创建服务实例
mc::dict config = {{"host", "localhost"}, {"port", 5432}};
auto db_service = factory.create_service("database", "main_db", config);

// 检查服务类型
if (factory.has_service("metrics")) {
    // 创建metrics服务
}

// 获取所有服务类型
auto service_types = factory.get_service_types();
```

## 6. 服务类型设计要求

要创建兼容的服务类型，需要满足以下条件：

1. 继承自`mc::service`基类
2. 提供接受`std::string`参数的构造函数(服务名称)
3. 实现`bool init(mc::dict&& args)`方法用于初始化
4. 提供静态`register_options`方法用于注册服务选项(可选)

## 7. 与其他组件的关系

服务工厂通常与以下组件协同工作：

- **Plugin Manager**：插件加载后向服务工厂注册服务类型
- **Service Manager**：使用服务工厂创建和管理服务实例
- **Configuration Manager**：提供服务创建所需的配置参数
