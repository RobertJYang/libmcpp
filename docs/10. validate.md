# mc::validate - C++ 类型校验库

## 概述
`mc::validate` 是一个轻量级的 C++ 类型校验库，专门用于在类型转换前进行整数类型的类型和范围检查。该库采用编译时类型检查和运行时范围验证相结合的方式，确保类型转换的安全性。

## 特性
- 🔒 **类型安全**：在编译时和运行时检查类型一致性
- 🚀 **零额外开销**：使用模板和内联，在大多数情况下没有运行时性能损失
- 🎯 **精确的范围检查**：正确处理有符号/无符号整数转换边界情况
- 🛡️ **异常安全**：提供详细的异常信息，便于调试
- 📦 **头文件库**：仅包含头文件，易于集成

## 异常类型
库定义了三种异常类型，均继承自 `std::runtime_error`：

| 异常类型        | 描述           | 触发条件                              |
|-----------------|----------------|---------------------------------------|
| `TypeError`     | 类型错误       | 传入值的类型不是整数类型              |
| `RangeError`    | 范围错误       | 传入值超出目标类型的表示范围          |
| `InternalError` | 内部错误       | 不支持的目标类型（目前仅支持整数）    |

## API 参考

### Validator 类
```cpp
namespace mc::validate {
    class Validator {
    public:
        template<typename T, typename ValueType>
        void check(ValueType&& value) const;
    };
}
```
# 方法说明

## `check<T>(ValueType&& value)`

### 模板参数：
- **T**：目标类型（必须是整数类型）
- **ValueType**：传入值的类型（自动推导）

### 参数：
- **value**：要检查的值（支持右值引用和完美转发）

### 异常：
- 抛出 `TypeError`：如果 `value` 的基础类型不是整数类型
- 抛出 `RangeError`：如果 `value` 超出目标类型 `T` 的表示范围
- 抛出 `InternalError`：如果目标类型 `T` 不是整数类型（目前仅支持整数）

### 使用示例

#### 基本使用

```cpp
#include "validate.h"
#include <iostream>

int main() {
    mc::validate::Validator validator;
    try {
        int64_t large_value = 5000000000LL; // 检查是否可转换为 int32_t
        validator.check<int32_t>(large_value); // 抛出 RangeError
    } catch (const mc::validate::RangeError& e) {
        std::cout << "范围错误: " << e.what() << std::endl;
    } catch (const mc::validate::TypeError& e) {
        std::cout << "类型错误: " << e.what() << std::endl;
    }
    return 0;
}
```

目前支持的类型
======================
库支持所有标准 C++ 整数类型：
- 类型 说明
  - `int8_t, int16_t, int32_t, int64_t` 有符号整数
  - `uint8_t, uint16_t, uint32_t, uint64_t` 无符号整数
  - `char, signed char, unsigned char` 字符类型（被视为整数）
  - `short, int, long, long long` 平台相关有符号整数
  - `unsigned short, unsigned int, unsigned long, unsigned long long` 平台相关无符号整数

  #### 完整示例
```cpp
#include "validate.h"
#include <iostream>
#include <vector>

void test_validation() {
    mc::validate::Validator validator;

    // 测试用例
    std::vector<std::pair<const char*, std::function<void()>>> tests = {
        {"有效转换 int16_t -> int32_t", []() {
            int16_t value = 100;
            validator.check<int32_t>(value); // 成功
            std::cout << "✓ int16_t(100) 可转换为 int32_t" << std::endl;
        }},
        {"有符号转无符号（正值）", []() {
            int32_t value = 255;
            validator.check<uint8_t>(value); // 成功
            std::cout << "✓ int32_t(255) 可转换为 uint8_t" << std::endl;
        }},
        {"有符号转无符号（负值）", []() {
            int32_t value = -1;
            try {
                validator.check<uint8_t>(value); // 抛出 RangeError
            } catch (const mc::validate::RangeError& e) {
                std::cout << "✓ int32_t(-1) 无法转换为 uint8_t: " << e.what() << std::endl;
            }
        }},
        {"大值转小类型", []() {
            int64_t value = 1000;
            try {
                validator.check<int8_t>(value); // 抛出 RangeError
            } catch (const mc::validate::RangeError& e) {
                std::cout << "✓ int64_t(1000) 无法转换为 int8_t: " << e.what() << std::endl;
            }
        }},
        {"无符号转有符号（在范围内）", []() {
            uint16_t value = 32767;
            validator.check<int16_t>(value); // 成功
            std::cout << "✓ uint16_t(32767) 可转换为 int16_t" << std::endl;
        }},
        {"无符号转有符号（超出范围）", []() {
            uint16_t value = 65535;
            try {
                validator.check<int16_t>(value); // 抛出 RangeError
            } catch (const mc::validate::RangeError& e) {
                std::cout << "✓ uint16_t(65535) 无法转换为 int16_t: " << e.what() << std::endl;
            }
        }},
    };

    for (const auto& [name, test_func] : tests) {
        std::cout << "\n测试: " << name << std::endl;
        try {
            test_func();
        } catch (const std::exception& e) {
            std::cout << "✗ 未预期的异常: " << e.what() << std::endl;
        }
    }
}

int main() {
    test_validation();
    return 0;
}
```
